# Spec: Display Error Message for Invalid Hexadecimal Color Codes (#9527)

## Goal
Detect incorrect hexadecimal color codes in user input and display a clear error message, to improve user experience and prevent confusion.

## Context
- Issue: #9527
- Current behavior: The input is ignored silently and color remains unchanged. No error or feedback is shown to the user.
- Expected: A clear and visible error message should appear when an invalid hexadecimal value is entered

## Component / Module
- `packages/excalidraw/components/ColorPicker/ColorInput.tsx` (hex input handling and validation trigger)
- `packages/common/src/colors.ts` (`normalizeInputColor` validation and normalization logic)
- `packages/excalidraw/components/ColorPicker/ColorPicker.tsx` (hosts `ColorInput` in the color picker flow)
- `packages/excalidraw/tests/colorInput.test.ts` (primary validation behavior tests)
- `packages/common/src/colors.test.ts` (supporting color utility tests)

## Non-goals
- Do NOT change supported color formats outside the hex color input flow
- Do NOT change eye dropper behavior
- Do NOT refactor unrelated color picker UX
- Do NOT add new dependencies for validation or error handling
- Do NOT create a separate error component; use existing styling and patterns for error messages in the app
- Do NOT implement auto-correction or suggestions for invalid hex codes; only display an error message
- Do NOT add localization for other languages at this time; only ensure that error messages are set up for future localization

## Acceptance Criteria
1. Entering an invalid hex length shows: "Hex code must be 3, 4, 6, or 8 characters (excluding #)"
2. Entering non-hex characters shows: "Invalid characters in hex code"
3. Entering a malformed hex value shows: "Not a valid hex color"
4. Entering a valid hex value applies the color and clears any previous error
5. Error messages are styled consistently with the app's design system and are clearly visible
6. Error messages supporting localization (i18n) with appropriate keys and translations
7. Existing tests pass (`yarn test:app`)
8. Build passes (`yarn build`)

## Edge Cases
- Treat repeated `#` characters as formatting noise and remove them before hex validation.
- Remove spaces and tab characters from any position in the input before validation.
- If the input is cleared and left empty, keep the previously applied color and do not show an error.
